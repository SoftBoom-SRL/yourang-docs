# Complete glossary

All terms used in Yourang explained clearly, without jargon.

_This glossary collects the terms you'll encounter using Yourang, explained simply and directly. If a term is missing, you can report it via the support channel._

> [!NOTE]
> **How to use the glossary**
> Terms are in alphabetical order. If you're reading the documentation and come across a word you don't know, look it up here.

**Voice agent**
: The automated assistant that answers phone calls instead of your staff. You can configure its name, voice, and the tasks it is authorized to perform.

**Alert**
: An automatic notification sent to your team when an important event occurs, such as an urgent booking or an after-hours request requiring human attention.

**AMD (Answering Machine Detection)**
: Feature that automatically detects whether an outgoing call was answered by a real person or by an answering machine, avoiding leaving useless messages.

**Voice preview**
: Feature that lets you listen to how the assistant will sound before making changes live, so you can verify tone, pronunciation, and naturalness of the voice.

**Custom field**
: Additional information you can associate with a contact, such as room preference or spoken language. Custom fields let you enrich the customer profile with data specific to your property.

**Incoming call**
: A phone call received from your customers. The assistant answers, handles the conversation, and, if necessary, transfers the call to an operator.

**Outgoing call**
: A phone call started by the system toward a customer, for example for a recall campaign, a booking confirmation, or a reminder.

**Confirmation**
: Message sent to the customer to communicate that a booking, change, or request has been received and correctly recorded.

**Criterion**
: A rule that defines how the assistant evaluates the outcome of a call, for example whether the customer accepted an offer, booked, or asked to be contacted again.

**Outcome**
: The result recorded at the end of a call: confirmed booking, information request, transfer to operator, no answer, and so on.

**Filter**
: Dashboard tool that lets you view only the data you're interested in, for example calls from a certain date or contacts with a specific status.

**Integration**
: Connection between Yourang and an external service, such as the property's management system, calendar, or online booking system. Integrations allow data to sync automatically.

**Price list**
: The list of rates, services, and availability that the assistant can consult and communicate to customers during calls.

**Operator**
: A member of your staff who can intervene in a call handled by the assistant, receive transfers, or monitor conversations in real time.

**Scope**
: The set of tasks and topics the assistant is authorized to handle. Anything beyond the scope is routed to a human operator.

**Booking**
: The request from a customer to reserve a service (room, table, space). The assistant can collect bookings and record them in the system.

**Summary**
: The automatic report generated at the end of every call, which includes the main points of the conversation and the actions taken by the assistant.

**Slot**
: A time interval available for a booking or appointment. The assistant checks slot availability and assigns them based on customer requests.

**Call status**
: The indicator that shows the current phase of a call: waiting, in progress, transferred, completed, or unanswered.

**Transcript**
: The text version of a phone conversation, automatically generated after each call. You can consult it from the dashboard to verify what was said.

**Transfer**
: The operation by which the assistant passes the call to a human operator, for example when the customer's request is complex or outside the configured scope.

**Webhook**
: A technical mechanism that automatically sends a notification to an external system when an event occurs in Yourang, such as a new booking or the end of a call.
